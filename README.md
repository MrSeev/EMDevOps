## Требования
- Docker Engine 
- Git (Опционально)

## Запуск
- Склонировать или скачать репозиторий с github
`git clone https://github.com/MrSeev/EMDevOps.git`
- Перейти в терминале в директорию проекта **EMDevOps**
`cd EMDevOps`
- Запустить команду сборки и запуска docker compose
`docker compose up --build -d`

## Проверка работоспособности запущенного веб сервера и backend сервиса
- В браузере перейти по адресу http://localhost
    Откроется вывод формата json "Hello from Effective Mobile!"
- Ввести в терминале команду
    `curl http://localhost`
    Терминал выдаст сообщение "Hello from Effective Mobile!"

## Описание проекта (Архитектура)
Проект использует основу Docker Compose для развертывания двух сервисов: backend и nginx
Backend:
    - Используется Dockerfile для сборки образа сервиса
    - Сервис построен на базе образа python:3.14.4-slim
    - Для создания простого HTTP-сервера используется связка FastAPI и Uvicorn
    - Файл requirements.txt содержит необходимые pip зависимости, которые устанавливаются при сборке образа
    - Скрипт app.py является точкой входа для запуска HTTP-сервера, а также содержит маршрут /
    - На сервисе открыт порт 8080 только внутри сети docker
    - Сервис подключен к общей сети docker_network
Nginx:
    - Сервер построен на базе образа nginx:1.29
    - Кастомная конфигурация сервера nginx.conf монтируется из директории nginx в контейнер через volume
    - На сервере открыт порт 80 для хостовой машины
    - Запуск сервиса nginx зависит от успешного запуска сервиса backend
    - Сервер nginx проксирует входящее соединение, перенаправляя его на внутренний порт 8080 сервиса backend
    - Сервис подключен к общей сети docker_network
В проекте используется кастомная сеть docker_network с драйвером bridge
Эта архитектура позволяет изолировать сервисы в контейнерах, при этом Nginx выполняет роль прокси-сервера, который обрабатывает запросы и перенаправляет их

## Список использованных технологий
- Python - язык программирования для разработки backend-сервиса
- FastApi - фреймворк для создания API, используется для создания HTTP-сервера
- Uvicorn - ASGI-сервер для FastAPI, который обрабатывает запросы
- Nginx - прокси-сервер для обработки и перенаправления HTTP-запросов на backend
- Docker - система контейнеризации для создания и управления сервисами в изолированных контейнерах
- Docker Compose - инструмент для управления многоконтейнерными Docker-приложениями, используемый для упрощения настройки и запуска сервисов
